	<h1 class="firstHeading" id="firstHeading" lang="en">NoSQLz</h1>
	
	<div class="mw-body-content" id="bodyContent">

		<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="en"><div class="mw-parser-output"><table class="infobox vevent" style="width:22em"><caption class="summary">NoSQLz</caption><tbody><tr><th scope="row" style="white-space: nowrap;">Initial release</th><td>2014</td></tr><tr style="display: none;"><td colspan="2" style="text-align:center"></td></tr><tr><th scope="row" style="white-space: nowrap;"><a href="/wiki/Software_release_life_cycle" title="Software release life cycle">Stable release</a></th><td><div style="margin:0px;">2.1.0

<p><b>NoSQLz</b> is a consistent key-value <a href="/wiki/Big_data" title="Big data">big data</a> store (<a href="/wiki/NoSQL" title="NoSQL">NoSQL</a> database) for <a href="/wiki/Z/OS" title="Z/OS">z/OS</a> <a href="/wiki/IBM" title="IBM">IBM</a> systems.<sup class="reference" id="cite_ref-1"><a href="#cite_note-1">[1]</a></sup> It was developed by systems programmer Thierry Falissard in 2013. The purpose was to provide a low-cost alternative to all proprietary <a class="mw-redirect" href="/wiki/Mainframe" title="Mainframe">mainframe</a> <a class="mw-redirect" href="/wiki/DBMS" title="DBMS">DBMS</a> (version 1 is <a href="/wiki/Free_software" title="Free software">free software</a>).
</p>
<h2><span class="mw-headline" id="Distinctive_features">Distinctive features</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=NoSQLz&amp;action=edit&amp;section=1" title="Edit section: Distinctive features">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>NoSQLz only provides basic <a href="/wiki/Create,_read,_update_and_delete" title="Create, read, update and delete">create, read, update and delete</a> (CRUD) functions. It is designed to be very straightforward and easy to implement.
</p><p><a href="/wiki/ACID" title="ACID">ACID</a> properties are provided, so as to have "real transactions", through <a href="/wiki/Optimistic_concurrency_control" title="Optimistic concurrency control">optimistic concurrency control</a>, <a href="/wiki/Timestamp-based_concurrency_control" title="Timestamp-based concurrency control">timestamp-based concurrency control</a> and <a href="/wiki/Multiversion_concurrency_control" title="Multiversion concurrency control">multiversion concurrency control</a> (MVCC).
</p>
<h2><span class="mw-headline" id="Interfaces">Interfaces</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=NoSQLz&amp;action=edit&amp;section=2" title="Edit section: Interfaces">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Unlike version 1, version 2 of NoSQLz is chargeable and supports <a href="/wiki/IBM_Parallel_Sysplex" title="IBM Parallel Sysplex">IBM Parallel Sysplex</a>. The NoSQLz DBMS can be interfaced in <a href="/wiki/Rexx" title="Rexx">Rexx</a>, <a class="mw-redirect" href="/wiki/Cobol" title="Cobol">Cobol</a>, <a href="/wiki/IBM_High_Level_Assembler" title="IBM High Level Assembler">IBM High Level Assembler</a>, etc.
</p>
