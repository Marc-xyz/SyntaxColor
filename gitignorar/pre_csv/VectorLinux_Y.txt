<p><b>VectorLinux</b>, abbreviated VL, is a <a href="/wiki/Linux_distribution" title="Linux distribution">Linux distribution</a> for the <a href="/wiki/X86" title="X86">x86</a> platform based on the <a href="/wiki/Slackware" title="Slackware">Slackware</a> Linux distribution, originally developed by Canadian developers Robert S. Lange and Darell Stavem. Since version 7 the Standard Edition is also available for the <a href="/wiki/X86-64" title="X86-64">x86-64</a> platform, known as VLocity64 7.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup>
</p>

<h2><span class="mw-headline" id="Editions">Editions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=VectorLinux&amp;action=edit&amp;section=1" title="Edit section: Editions">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Four editions of VectorLinux are maintained: SOHO, Standard, Light and Live.
</p>
<h3><span id="SOHO_Edition_.28Discontinued_after_version_7.0.29"></span><span class="mw-headline" id="SOHO_Edition_(Discontinued_after_version_7.0)">SOHO Edition (Discontinued after version 7.0)</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=VectorLinux&amp;action=edit&amp;section=2" title="Edit section: SOHO Edition (Discontinued after version 7.0)">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The SOHO (Small Office/Home Office) Edition is designed for modern computers and is based on the <a href="/wiki/KDE" title="KDE">KDE</a> desktop environment. Applications include <a href="/wiki/LibreOffice" title="LibreOffice">LibreOffice</a>, <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a>, the <a href="/wiki/GIMP" title="GIMP">GIMP</a>, <a class="mw-redirect" href="/wiki/Xsane" title="Xsane">Xsane</a>, <a href="/wiki/CUPS" title="CUPS">CUPS</a>, <a class="mw-redirect" href="/wiki/Xara_Xtreme" title="Xara Xtreme">Xara Xtreme</a>, financial applications, and others. A deluxe edition of SOHO, which contains additional applications, is also available.
</p>
<h3><span class="mw-headline" id="Standard_Edition">Standard Edition</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=VectorLinux&amp;action=edit&amp;section=3" title="Edit section: Standard Edition">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Standard Edition is a freely downloadable distribution based on <a href="/wiki/Xfce" title="Xfce">Xfce</a> and specifically designed for use on older computers with slower processors and less RAM. The Deluxe Standard Version contains the standard edition plus numerous additional applications specific to that VL version. The CD includes many extra applications which you can add to build the system which best fits your requirements. Additional included applications are Libreoffice and GIMP among many others.
</p>
<h3><span class="mw-headline" id="Light_Edition">Light Edition</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=VectorLinux&amp;action=edit&amp;section=4" title="Edit section: Light Edition">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The Light Edition is designed for older computers with limited resources. Version 7.2 is based upon the <a href="/wiki/IceWM" title="IceWM">IceWM</a> window manager. It is supplied with Seamonkey as the browser/email client/html editor and includes some of the applications found in the Standard Edition. Additional applications are available through the package manager or by compiling user applications. Light Edition is usable on computers with 256 MB of RAM.
</p>
<h3><span class="mw-headline" id="Live_Edition">Live Edition</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=VectorLinux&amp;action=edit&amp;section=5" title="Edit section: Live Edition">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The Live Editions are <a class="mw-redirect" href="/wiki/LiveCD" title="LiveCD">bootable</a> CDROMs which contain the VectorLinux operating system on the CD. This enables the user to test a distribution on a system without the need of re-partitioning and installing to the hard drive. It is also possible to install VectorLinux from the Live Edition CD. VectorLinux presently has two Live Editions: Standard and SOHO. The latest edition is a preview of the Deluxe Edition.
</p><p>The <a href="/wiki/SuperGamer" title="SuperGamer">SuperGamer</a> distribution is based on the <i>VectorLinux</i> code base.
</p><p>Version 7.1 does not have a live edition. 
</p><p>Version 7.2 Standard (XFCE) has a live edition. Version 7.2 Light (IceWM) does not have a live edition. 
</p>
<h2><span class="mw-headline" id="Features">Features</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=VectorLinux&amp;action=edit&amp;section=6" title="Edit section: Features">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>One prominent feature that all VL versions have in common is the standard installation of system administration applications: VasmCC handles system configuration, while slapt-get with its GUI front-end Gslapt manages software installation and removal.
</p><p>VasmCC stands for 'Vector Administrative and Services Menu Control Center', and it configures everything from disk partitions to <a href="/wiki/X.Org_Server" title="X.Org Server">X.Org Server</a>. In addition to offering a <a class="mw-redirect" href="/wiki/GUI" title="GUI">GUI</a> interface, a text mode parallel to the GUI VasmCC is available. VasmCC has been available since Vector 2.0 was released and has been upgraded continually over the years, with the addition of GUI functionality in SOHO 3.2. Configuration tools like <a class="new" href="/w/index.php?title=Netconfig&amp;action=edit&amp;redlink=1" title="Netconfig (page does not exist)">netconfig</a> and <a class="new" href="/w/index.php?title=Alsaconf&amp;action=edit&amp;redlink=1" title="Alsaconf (page does not exist)">alsaconf</a> are also available in Vector.
</p><p>Gslapt is a GUI front-end to the <a href="/wiki/Slapt-get" title="Slapt-get">slapt-get</a> software management tool. Combined with lzma-compression and dependency tracking, the inclusion of Gslapt offers VectorLinux users the ability to quickly install and remove software while avoiding so-called '<a href="/wiki/Dependency_hell" title="Dependency hell">dependency hell</a>.' File compression via <a class="mw-redirect" href="/wiki/Lzma" title="Lzma">lzma</a> allows low and high bandwidth users alike to minimize download times.
</p>
<h3><span class="mw-headline" id="Package_management">Package management</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=VectorLinux&amp;action=edit&amp;section=7" title="Edit section: Package management">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Two graphical frontends have been written for <a href="/wiki/Slapt-get" title="Slapt-get">slapt-get</a>, the backwards-compatible dependency-tracking extension to Slackware's package tools used by VectorLinux. VL-APT was the first, written originally by the Vector developers to serve the need for GUI package installation and management, while gslapt was written by the original developers of slapt-get for the same purpose. Until recently, either option has been viable; however, recent changes in the slapt-get packaging format have made VL-APT obsolete, and the developers of Vector have consequently promoted gslapt as the preferred method of graphical package management.
</p><p>VL-APT (now deprecated) and gslapt are configured by default to use Vector's repository, which offers the majority of most users' needs. However, alternative sources, including mirrors of the official archives around the world, can be added through their respective configuration menus, or directly, through the textual slapt-getrc configuration. There is criticism, at least partially justified, regarding the size of the Vector package selection. These deficiencies have stimulated significant volunteer and developer response, and many desired software packages have now entered the Vector repository.
</p><p>In fact, it is not uncommon to use the information in /etc/slackware-version in Vector to download and install Slackware packages from LinuxPackages(dot)net, a common <a href="/wiki/Slackware" title="Slackware">Slackware</a> repository, GnomeSlackBuild, etc.<sup class="reference" id="cite_ref-Slackware_Packages_4-0"><a href="#cite_note-Slackware_Packages-4">[4]</a></sup>
</p><p>Vector Linux uses .tlz packages (LZMA-packed), though .tbz and .tgz are supported too. Pkgtools are taken from Tukaani Linux project and are non-standard - there are improvements over standard pkgtools in speed of operation, but have not been updated for a while.
</p>
